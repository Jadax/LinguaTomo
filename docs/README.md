# Documentation index

The documentation is split by decision boundary so humans and LLM agents can
load the minimum useful context.

| Document | Use it for |
|---|---|
| `../README.md` | Product overview, setup and release status |
| `ARCHITECTURE.md` | Module ownership, data flow and integration boundaries |
| `CURRICULUM.md` | Progression, pedagogy, assessment and content quality |
| `CONTENT.md` | Source audit, licences, attribution and imports |
| `DEVELOPMENT.md` | Toolchain, configuration, builds, signing and release steps |
| `QUALITY.md` | Test matrix, known warnings and troubleshooting |
| `ROADMAP.md` | Feature gaps, priorities and acceptance criteria |
| `../THIRD_PARTY_CONTENT.md` | Exact bundled third-party provenance |
| `../AGENTS.md` | Compact instructions for coding agents |

Keep each document focused. Link to another document instead of duplicating
large sections. Store dated research conclusions in `CONTENT.md`, not in source
code comments.
